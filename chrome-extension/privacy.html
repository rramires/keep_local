📝 KeepLocal

Last updated: March 2026 · Última atualização: Março 2026

Summary: KeepLocal does not collect, transmit, or store any data outside your browser. All your data is encrypted locally and never leaves your device.

1. Data Collection

KeepLocal does not collect any personal data. Specifically:

  • No information is sent to external servers
  • No analytics or tracking is used
  • No cookies are created
  • No network requests are made by the extension

2. Local Storage

All data is stored exclusively in your browser's IndexedDB, on your device. Data includes:

  • Notes and tasks (encrypted)
  • Theme preference (dark/light)
  • Cryptographic salt and password verifier

3. Encryption

All note and task data is encrypted using:

  • AES-256-GCM — military-grade symmetric encryption
  • PBKDF2 with 600,000 iterations and SHA-256 for key derivation
  • Random 12-byte IV generated for each encryption operation
  • Web Crypto API native to the browser (no external cryptography libraries)

4. Password

Your password is never stored in plain text. Only a PBKDF2-derived hash is saved for verification. There is no password recovery — if you forget your password, your data cannot be recovered.

5. Permissions

KeepLocal does not request any permissions from Chrome. It works entirely with standard browser APIs (IndexedDB, Web Crypto API, sessionStorage).

6. Open Source

The complete source code is available at github.com/rramires/keep_local under the MIT license. Anyone can audit the code to verify these privacy practices.

7. Contact

For privacy questions, open an issue on the GitHub repository.

Resumo: O KeepLocal não coleta, transmite ou armazena nenhum dado fora do seu navegador. Todos os seus dados são criptografados localmente e nunca saem do seu dispositivo.

1. Coleta de Dados

O KeepLocal não coleta nenhum dado pessoal. Especificamente:

  • Nenhuma informação é enviada para servidores externos
  • Nenhum analytics ou rastreamento é utilizado
  • Nenhum cookie é criado
  • Nenhuma requisição de rede é feita pela extensão

2. Armazenamento Local

Todos os dados são armazenados exclusivamente no IndexedDB do seu navegador, no seu dispositivo. Os dados incluem:

  • Notas e tarefas (criptografadas)
  • Preferência de tema (claro/escuro)
  • Salt criptográfico e verificador de senha

3. Criptografia

Todos os dados de notas e tarefas são criptografados usando:

  • AES-256-GCM — criptografia simétrica de grau militar
  • PBKDF2 com 600.000 iterações e SHA-256 para derivação de chave
  • IV aleatório de 12 bytes gerado para cada operação de criptografia
  • Web Crypto API nativa do navegador (sem bibliotecas externas de criptografia)

4. Senha

Sua senha nunca é armazenada em texto puro. Apenas um hash derivado via PBKDF2 é salvo para verificação. Não existe recuperação de senha — se você esquecer sua senha, os dados não poderão ser recuperados.

5. Permissões

O KeepLocal não solicita nenhuma permissão do Chrome. Ele funciona inteiramente com APIs padrão do navegador (IndexedDB, Web Crypto API, sessionStorage).

6. Código Aberto

O código-fonte completo está disponível em github.com/rramires/keep_local sob licença MIT. Qualquer pessoa pode auditar o código para verificar estas práticas de privacidade.

7. Contato

Para questões sobre privacidade, abra uma issue no repositório do GitHub.
