📝 KeepLocal — Política de Privacidade

Última atualização: Março 2026

Resumo: O KeepLocal não coleta, transmite ou armazena nenhum dado fora do seu navegador. Todos os seus dados são criptografados localmente e nunca saem do seu dispositivo.

1. Coleta de Dados

O KeepLocal não coleta nenhum dado pessoal. Especificamente:

  • Nenhuma informação é enviada para servidores externos
  • Nenhum analytics ou rastreamento é utilizado
  • Nenhum cookie é criado
  • Nenhuma requisição de rede é feita pela extensão

2. Armazenamento Local

Todos os dados são armazenados exclusivamente no IndexedDB do seu navegador, no seu dispositivo. Os dados incluem:

  • Notas e tarefas (criptografadas)
  • Preferência de tema (claro/escuro)
  • Salt criptográfico e verificador de senha

3. Criptografia

Todos os dados de notas e tarefas são criptografados usando:

  • AES-256-GCM — criptografia simétrica de grau militar
  • PBKDF2 com 600.000 iterações e SHA-256 para derivação de chave
  • IV aleatório de 12 bytes gerado para cada operação de criptografia
  • Web Crypto API nativa do navegador (sem bibliotecas externas de criptografia)

4. Senha

Sua senha nunca é armazenada em texto puro. Apenas um hash derivado via PBKDF2 é salvo para verificação. Não existe recuperação de senha — se você esquecer sua senha, os dados não poderão ser recuperados.

5. Permissões

O KeepLocal não solicita nenhuma permissão do Chrome. Ele funciona inteiramente com APIs padrão do navegador (IndexedDB, Web Crypto API, sessionStorage).

6. Código Aberto

O código-fonte completo está disponível em github.com/rramires/keep_local sob licença MIT. Qualquer pessoa pode auditar o código para verificar estas práticas de privacidade.

7. Contato

Para questões sobre privacidade, abra uma issue no repositório do GitHub.
